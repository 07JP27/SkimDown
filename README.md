# SkimDown

SkimDown is a lightweight macOS Markdown viewer for reading Markdown document sets generated by AI agents. It opens a folder, shows only Markdown files in a sidebar tree, and renders the selected file in a clean read-only preview.

## Documentation

The docs site is for end-user installation and usage. Developer workflows live in this README.

- [English docs](docs/index.md)
- [日本語ドキュメント](docs/ja/index.md)

## MVP features

- Open a folder with `File > Open Folder...`, `Cmd+O`, or folder drag-and-drop
- Multiple windows for multiple folders
- Recursive `.md` and `.markdown` discovery
- Hidden-file and ignored-directory filtering
- Folder-first, case-insensitive Markdown tree
- Read-only `WKWebView` preview
- Bundled Markdown rendering assets
- Current-document search
- Sidebar position, visibility, width, theme, font size, recent folders, last selected file, and tree expansion persistence
- File-system watching for Markdown add, delete, rename, and update events

## Architecture

SkimDown is a Swift 6 AppKit application. Markdown is rendered in a `WKWebView` with bundled JavaScript and CSS assets, so normal preview rendering does not depend on a CDN.

```text
src/
  project.yml
  SkimDown/
    App/
    Core/
    Sidebar/
    Markdown/
    Viewer/
    Models/
    Utilities/
    Resources/
  SkimDownTests/
```

## Development

### Prerequisites

- macOS 26 or later
- Xcode 26 or later
- XcodeGen
- Node.js and npm for the VitePress documentation site

### XcodeGen workflow

The Xcode project is generated from `src/project.yml`.

```bash
make generate
```

`make build`, `make test`, `make run`, `make release`, and `make launch-check` regenerate the project before using it.

### Build and run

```bash
make generate
make build
make run
```

### Tests

```bash
make launch-check
make test
```

`make launch-check` is a GUI smoke test for the failure mode unit tests cannot catch. It builds the app, launches the generated `.app`, activates SkimDown, verifies that a SkimDown window is on screen, and checks that bundled resources such as the app icon and Web renderer are present.

The unit test suite focuses on logic that can be validated reliably without UI automation: Markdown scanning, tree ordering, initial selection, link routing, folder-boundary checks, and settings persistence.

### Documentation site

```bash
make docs
make docs-build
```

`make docs` starts the local VitePress development server. `make docs-build` builds the static documentation site.

## Security model

SkimDown is sandboxed. It requests read-only access to user-selected folders, persists access using app-scoped security-scoped bookmarks, keeps local file references constrained to the opened folder, sanitizes embedded HTML, and opens external links in the default browser.

## Release and distribution

Release helpers are available through:

```bash
make release
make notarize
make dmg VERSION=1.0.0
```

Notarization requires `APPLE_ID`, `APPLE_TEAM_ID`, and `APPLE_APP_PASSWORD` environment variables.

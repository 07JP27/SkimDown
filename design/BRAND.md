# SkimDown — ブランドガイド

## ブランドの核

SkimDown は、AIエージェント、開発ツール、チームが生み出す Markdown の束を、静かに読み進めるための macOS リーダー。

エディタではなく、ドキュメント管理ツールでもなく、読むことに特化した「Markdown の読書面」を提供する。

## ブランドキーワード

- **Transparent paper**: 透明な紙片、重なり、軽さ。
- **Quiet focus**: 余白、静けさ、読むことへの集中。
- **Generated-doc flow**: 生成された仕様書、設計メモ、README、ログを流れのまま確認する。
- **Native macOS premium**: macOSらしい奥行き、ガラス感、磨かれた質感。
- **Read-only trust**: 誤編集しない、選んだフォルダだけを読む、外部へ送信しない。

## 訴求軸

### From scattered files to a reading surface

Markdown の束を、読むための画面に変える。フォルダを開くと Markdown だけが並び、選んだファイルを読みやすいプレビューで表示する。

### No editor chrome. No accidental edits.

エディタのペイン、保存確認、誤編集の不安を避ける。読む、移動する、表示中ファイルを検索することに絞る。

### Local, read-only, folder-bound

選んだフォルダだけを読み取り、Markdown 本文やフォルダ内容を外部へ送信しない。ローカルファイル参照も開いたフォルダ内に制限する。

### Built for generated documentation

仕様書、設計メモ、README、作業ログ、チェックリストなど、生成され続ける Markdown を短時間で俯瞰する。

## トーン&マナー

- 短いコピーで価値を言い切る。
- 過剰に技術用語を出さず、ユーザーの読み方を描写する。
- 「AI」だけに閉じず、agents / tools / teams が生む Markdown と表現する。
- 読み取り専用、ローカル、外部送信なしは信頼要素として明確にする。
- 余白を広く取り、透明な紙片・ガラスカード・淡い発光で軽さを出す。

## Do / Don't

### Do

- アイコン由来の淡い色と透明感を使う。
- hero は大きく、短く、余白を持たせる。
- 3ステップで使い方を伝える。
- ユーザー向け docs の責務を守る。

### Don't

- 開発者向け build コマンドをトップページに戻さない。
- Postman や Notion の見た目をそのままコピーしない。
- 万能 Markdown エディタのように見せない。
- まだ存在しない配布・機能・実績を訴求しない。
